# DeutschFlow — Curriculum Principles

**Version:** 1.0 · **Updated:** 2026-07-12 · **Status:** Active · **Sprint:** 6.3A

> How lessons chain into a CEFR journey. Governs sequencing, spiralling, and difficulty across A1→C1.

---

## 1. CEFR alignment

| Level | Real-world outcome | Units (target) | Lessons/unit |
|---|---|---|---|
| A1 Beginner | Introduce yourself, order food, ask directions, tell time | 3–4 | 4–6 |
| A2 Elementary | Travel, shopping, daily routines, simple past | 3–4 | 4–6 |
| B1 Intermediate | Opinions, work, media, relationships | 4 | 4–6 |
| B2 Upper-Int. | Argue, abstract topics, culture, science | 4 | 5–6 |
| C1 Advanced | Academic, literary, nuanced debate | 4 | 5–6 |

Each level = the CEFR "can-do" outcome. Each **Unit** is a themed chunk (~4–6 lessons) delivering a graspable milestone; each **Lesson** is one capability (LESSON_BLUEPRINT §1). *CEFR = chapters; Units = the pages you turn today.*

---

## 2. Sequencing rules

1. **Frequency-first vocabulary:** A1 draws from the top ~500 German words.
2. **Capability dependency, not lock:** later lessons *assume* earlier capabilities but are **not gated** (advanced learners jump ahead — Guided-not-Gated).
3. **One new grammar pattern per lesson**, introduced only when the dialogue naturally contains it.
4. **Spiral review (interleaving):** every lesson's warm-up + dialogue reuse ≥2 prior words; every unit's last lesson reuses the unit's patterns.
5. **Errorless → productive-failure gradient across levels:** A1 is heavily scaffolded; by B2 learners attempt before hints.

---

## 3. Interleaving & spacing across lessons

- **Within a unit:** vocabulary from lesson N re-appears in N+1..N+3 warm-ups (spacing).
- **Across units:** SM-2 schedules review independent of lesson order (retrieval practice).
- **Mixed practice:** review sessions interleave words from multiple lessons rather than blocking one lesson's set (interleaving beats blocking for retention).

---

## 4. Difficulty progression (decision table)

| Axis | Early A1 | Late A1 / A2 | B1+ |
|---|---|---|---|
| Instruction language | English | English | English→German |
| Dialogue speed/length | slow, 4–6 lines | 6–8 | 8–12 |
| Exercise type mix | recognition-heavy (MCQ) | + fill-blank | + translate/produce |
| Scaffolding (hints, suggestions) | maximal | moderate | minimal |
| Translation availability | always one-tap | one-tap | one-tap but discouraged |

---

## 5. Authoring workflow (so hundreds of lessons stay consistent)

1. Pick **one capability** + its unit + CEFR level.
2. Write the **dialogue first** (realistic, 2–3 speakers, only known + this-lesson vocabulary).
3. **Extract** 8–12 vocabulary items and **1** grammar pattern from the dialogue.
4. Write **3–5 exercises** covering each item/pattern once.
5. Write **1 speaking prompt** with 1–2 suggestion scaffolds.
6. Set the **spiral** (which prior words to reuse in the warm-up).
7. **Native-check** all German. **Run the 4 success questions** (LESSON_BLUEPRINT §2).

---

## 6. Assumptions & trade-offs
- **Assumption:** content grows fastest at A1 (persona "Sarah, zero German"); A1 depth is the top priority (matches PRODUCT_AUDIT).
- **Trade-off:** strict "one pattern per lesson" means more, shorter lessons — better for mobile + confidence, at the cost of a longer table of contents (mitigated by Unit grouping on the Learn page).
- **Edge case:** cultural/idiomatic content that resists CEFR bucketing → tag as a "culture" side-lesson, optional, non-blocking.
